# Deploy to Render

## Architecture

| Service | Render name | Plan | Role |
|---------|-------------|------|------|
| BFF API | `solanist-bff` | Starter | .NET API, MongoDB Atlas |
| Web app | `solanist-app` | Starter | Angular + nginx, proxies `/api/` → BFF |

Both services use **Starter** instances (512 MB RAM, always-on — no free-tier cold starts). Region: **Frankfurt** (close to EU Atlas/S3 `eu-west-1`).

The browser only talks to `solanist-app`. Nginx forwards `/api/v1/*` to the BFF using `BFF_UPSTREAM` (set automatically from the blueprint).

## CI/CD

| Stage | Where | What |
|-------|--------|------|
| **CI** | GitHub Actions ([`.github/workflows/ci.yml`](../.github/workflows/ci.yml)) | On every push/PR: `dotnet build`, `npm run build`, Docker image builds |
| **CD** | Render | Auto-deploy `main` when the repo is connected via Blueprint |

Flow:

1. Open a PR → CI runs; merge when green.
2. Push to `main` → CI runs again; Render builds and deploys both Docker services from [`render.yaml`](../render.yaml).

No deploy hooks or extra secrets are required if Render is linked to GitHub. Check deploy status in Render Dashboard → each service → **Events**.

## One-time setup

1. Push this repo to GitHub (default branch `main`).
2. In [Render Dashboard](https://dashboard.render.com) → **New** → **Blueprint**.
3. Connect the repo — Render reads [`render.yaml`](../render.yaml) and creates both **Starter** web services.
4. When prompted, set **`Mongo__ConnectionString`** on `solanist-bff` to your Atlas URI.
5. Set production secrets on `solanist-bff` (Dashboard → Environment):
   - `Auth__GoogleClientId`
   - `Paystack__*` (if using billing) — callback URL auto-sets from `Auth__AppBaseUrl` when unset/localhost
   - `S3__*` / `Email__*` / `WhatsApp__ApiKey` as needed
6. In **MongoDB Atlas** → Network Access → allow **`0.0.0.0/0`** (or Render egress IPs).

After first deploy:

7. **Google OAuth** → Authorized JavaScript origins must include **`https://solanist-app.onrender.com`** (not localhost).
8. On `solanist-bff`, confirm **`Auth__AppBaseUrl`** = `https://solanist-app.onrender.com` (blueprint links this automatically).
9. Optional: set **`Auth__AllowSelfSignup`** = `true` on Render if you want the public “Create an account” flow (local `.env` default; blueprint sets `false`).

## Environment variables

### `solanist-bff`

| Variable | Required | Notes |
|----------|----------|-------|
| `Mongo__ConnectionString` | Yes | Atlas connection string |
| `Mongo__DatabaseName` | No | Default `solanist` |
| `Auth__SecretKey` | Auto | Generated by blueprint |
| `PORT` | Auto | Set by Render |

### `solanist-app`

| Variable | Required | Notes |
|----------|----------|-------|
| `BFF_UPSTREAM` | Auto | Linked to BFF `RENDER_EXTERNAL_URL` |

## Local Docker (unchanged)

```bash
cp .env.example .env   # add Mongo URI
npm run deploy
```

App: http://localhost:8080 · BFF: http://localhost:8081

## Notes

- **Starter tier:** both services stay warm (unlike Render free tier ~30s cold starts).
- **Cold start:** Mongo seed runs on first BFF boot if collections are empty.
- **Secrets:** never commit `.env` or Mongo credentials to git.
- **Smoke tests:** after deploy run `npm run smoke` (local) or `SMOKE_BASE_URL=https://solanist-app.onrender.com npm run smoke` — see [PRODUCTION.md](./PRODUCTION.md).
- **CI failures:** open the Actions tab on GitHub; fix build errors before merging to `main` (Render only deploys passing commits from connected branch).
